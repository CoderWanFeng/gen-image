# 短视频封面生成规范

本文档总结使用火山引擎 ARK `doubao-seedream-5-0` 模型生成短视频封面的标准流程与质量要求，后续所有封面任务均参照此规范执行。

---

## 1. 技术选型

- **模型**: `doubao-seedream-5-0-260128`（火山引擎 ARK）
- **SDK**: `volcenginesdkarkruntime`
- **接入方式**: 图生图（img2img），传入**多张人物参考图** + 文本 prompt
- **人物面部参考**: `assets/face/` 目录下**全部图片**（多角度多表情，提升面部一致性）
  - `Codex封面_3比4.png`：主参考图，包含整体风格（黑Polo衫 + 工作室布光）
  - `微信图片_*.jpg`（5 张）：本人不同角度/表情的素材照，作为面部特征辅助参考
  - 生图时将所有图片一起传入 `image` 参数，模型会综合参考所有图的面部特征
- **风格基准**: `tasks/gaokao/cover_gaokao_4x3.png`（已落地的目标风格成品图）
- **API Key**: 通过环境变量 `ARK_API_KEY` 注入
- **水印**: `watermark=False`

## 2. 任务目录结构

每个封面任务在 `tasks/` 下独立建一个子目录：

```
tasks/<任务名>/
├── quest.md                       # 需求描述（文案、情绪、动作等）
├── gen_cover_<任务名>.py           # 主生成脚本（一次跑出 3:4 / 4:3 / 16:9 / 2.35:1 四种比例）
├── cover_<任务名>_3x4.png          # 3:4 输出（竖版，抖音/小红书）
├── cover_<任务名>_4x3.png          # 4:3 输出（横版，公众号内文）
├── cover_<任务名>_16x9.png         # 16:9 输出（横版，视频号/B站/YouTube）
└── cover_<任务名>_2_35x1.png       # 2.35:1 输出（横版，公众号头条封面）
```

**封面文件命名规范**：

| 比例 | 文件名格式 | 用途 |
|------|----------|------|
| 3:4 | `cover_<任务名>_3x4.png` | 抖音/小红书竖版主封面 |
| 4:3 | `cover_<任务名>_4x3.png` | 公众号/横屏短视频 |
| 16:9 | `cover_<任务名>_16x9.png` | 视频号/B站/YouTube |
| 2.35:1 | `cover_<任务名>_2_35x1.png` | 公众号头条封面（横版，900×383 等比放大） |

- 比例标识用**下划线**与任务名分隔（`_3x4`），避免与任务名黏在一起
- 比例标识内部用**小写字母 x** 表示"比"（如 `3x4`、`4x3`、`16x9`），**不要用冒号**

**关键路径**：脚本中 `BASE_DIR` 必须指向项目根目录，正确写法：

```python
BASE_DIR = os.path.dirname(os.path.dirname(os.path.dirname(os.path.abspath(__file__))))
```

（脚本位于 `tasks/<任务名>/xxx.py`，需要向上回退 3 层）

## 3. 视觉风格标准（统一基准）

**风格基准成品**：`assets/refs/cover_gaokao_4x3.png`（即本次高考 4:3 封面）。所有后续封面必须与该图保持 100% 风格一致。

| 维度 | 标准 |
|------|------|
| **人物面部** | 必须与 `assets/face/` 目录下全部照片的面部特征完全匹配：戴黑金色细框方形眼镜、右脸颊下侧小痣、眉毛/眼睛/鼻子/嘴巴/脸型完全匹配，参考该目录下 5 张照片的多角度面部特征 |
| **面部表情参考** | `assets/face/` 目录下的 5 张照片提供了完整的表情基准（微笑、严肃、思考、点赞、指向上方），生图时表情需与这些照片的五官肌肉运动规律一致，避免出现不符合本人面部习惯的怪异表情 |
| **发型** | 黑色背头（头发向后梳，露出额头，两侧短），**不要炸毛/刺猬头** |
| **服装** | 黑色短袖 Polo 衫，领口两颗纽扣，面料有轻微褶皱质感 |
| **布光** | 工作室专业布光：面部光线均匀柔和，人物边缘有白色轮廓光，背景全黑 |
| **构图** | 人物居中偏右，大头像特写，高对比度，情绪张力强 |
| **画质** | 高清写实，皮肤细节真实，边缘锐利，无 AI 伪影 |
| **表情基调** | 冷笑/笃定/略带轻蔑，嘴角微抿，眼神直视镜头（可根据需求调整，调整范围不超出 `assets/face/` 中已有的表情风格） |

## 4. 文字标准（核心规范）

模型可直接在画面内生成中文标题，**无需后期合成**。文字规范如下：

| 维度 | 第一行标题 | 第二行标题 |
|------|----------|----------|
| 颜色 | 纯白色 `#FFFFFF` | 亮黄色 `#F5B800`（参考"必须用"配色） |
| 字体 | 粗体无衬线（类似"Codex"字体风格） | 粗体无衬线 |
| 字号 | 超大，占据画面顶部 | 超大，位于第一行下方 |
| 对齐 | 居中或左对齐 | 与第一行保持一致 |
| 质量要求 | 笔画工整、字形准确、无错字、无变形、无乱码 | 同左 |

> 在 prompt 中必须明确要求"文字必须清晰、笔画工整、字形准确，无错字、无变形、无乱码"，否则模型可能输出错字。

## 4.1 品牌与产品标识

若口播稿封面文案、创作背景、品牌展现或画面要求中涉及具体产品、企业、大厂名称，应在画面中加入对应产品/企业 logo 或可识别标识。

- 标识应作为辅助信息出现，不能遮挡标题文字、人物面部和核心手势
- 优先放在标题下方、画面侧边或人物旁边的留白区域
- 若无法准确生成官方 logo，可使用简洁图标 + 产品/企业名称文字组合替代
- 标识文字必须清晰准确，不能出现错字、变形、乱码
- 涉及多个品牌/产品时，按口播稿的核心对比关系分组呈现，避免堆满画面

## 5. 输出比例与尺寸

`doubao-seedream-5-0` 要求图像总像素 ≥ 3,686,400，常用尺寸：

| 比例 | 推荐尺寸 | 用途 |
|------|---------|------|
| 3:4 | `2K`（默认） | 抖音/小红书竖版主封面 |
| 4:3 | `2560x1920` | 公众号/横屏短视频 |
| 16:9 | `2560x1440` | 视频号/B站/YouTube |
| 2.35:1 | `3072x1307` | 公众号头条封面（≥ 3.69M 像素） |

⚠️ 不要使用低于 3,686,400 像素的尺寸（如 `2048x1536`），会被 API 拒绝。

## 6. Prompt 模板

新封面在此模板基础上替换文案、情绪、动作即可。**风格基准固定**，不要随意改动第 1、2、5、6 项：

```
仿照参考图片的视频封面风格，生成一个新的封面图片。整体风格必须与 cover_gaokao_4x3.png 完全一致。
要求：
1. 人物面部特征与所有参考图（assets/face/ 下 5 张多角度照片）完全一致：戴黑金色细框方形眼镜，右脸颊下侧有一颗小痣，黑色背头（头发向后梳，露出额头，两侧短），不要炸毛/刺猬头；五官形状、脸型、皮肤质感必须严格还原本人特征
2. 人物穿着黑色短袖Polo衫，领口两颗纽扣
3. 动作：<根据需求填写，例：双手交叉抱在胸前，身体微微后仰，姿态松弛但气场强大>
4. 表情：<根据需求填写，例：冷笑，嘴角微抿，眼神直视镜头，笃定中带着一丝轻蔑>；表情必须符合参考图中本人的五官肌肉运动规律，不要出现怪异或不像本人的表情
5. 背景：纯黑色背景，人物周围有冷色调（淡蓝色）聚光灯，人物边缘有清晰的白色轮廓光，营造工作室舞台感
6. 整体风格：短视频爆款封面，高对比度，人物大头像特写居中偏右，画质高清写实，皮肤细节真实，无 AI 伪影
7. 在画面上方/左侧添加两行超大中文标题文字：
   - 第一行："<标题1>"，纯白色，粗体无衬线字体（类似"Codex"字体风格）
   - 第二行："<标题2>"，亮黄色 #F5B800，粗体无衬线字体
   - 字号超大，与人物形成强烈视觉冲击
   - 文字必须清晰、笔画工整、字形准确，无错字、无变形、无乱码
8. 整体排版参考短视频爆款封面：大标题文字 + 人物特写
```

## 7. 标准代码模板

**一次跑出 4 个比例（3:4 / 4:3 / 16:9 / 2.35:1）**：

```python
import os
import base64
import urllib.request
from volcenginesdkarkruntime import Ark

BASE_DIR = os.path.dirname(os.path.dirname(os.path.dirname(os.path.abspath(__file__))))
FACE_DIR = os.path.join(BASE_DIR, 'assets', 'face')
OUT_DIR = os.path.dirname(os.path.abspath(__file__))

os.environ['ARK_API_KEY'] = '<your-key>'

# 加载 assets/face 下所有图片作为参考，提升面部一致性
images = []
for fname in sorted(os.listdir(FACE_DIR)):
    if fname.lower().endswith(('.png', '.jpg', '.jpeg')):
        with open(os.path.join(FACE_DIR, fname), 'rb') as f:
            images.append(f'data:image/png;base64,{base64.b64encode(f.read()).decode()}')

client = Ark(base_url='https://ark.cn-beijing.volces.com/api/v3', api_key=os.environ['ARK_API_KEY'])

prompt = '''...（参考第 6 节模板）...'''

# 一次跑出 3:4 / 4:3 / 16:9 / 2.35:1 四种比例
ratios = [
    ('3:4',  '2K',          'cover_<任务名>_3x4.png'),
    ('4x3',  '2560x1920',   'cover_<任务名>_4x3.png'),
    ('16x9', '2560x1440',   'cover_<任务名>_16x9.png'),
    ('2.35x1', '3072x1307', 'cover_<任务名>_2_35x1.png'),  # 公众号头条封面
]

for ratio_name, size, fname in ratios:
    print(f'正在生成 {ratio_name} ...')
    resp = client.images.generate(
        model='doubao-seedream-5-0-260128',
        prompt=prompt,
        image=images,  # 同样传入 assets/face 下全部参考图
        response_format='url',
        size=size,
        watermark=False
    )
    url = resp.data[0].url
    output_path = os.path.join(OUT_DIR, fname)
    urllib.request.urlretrieve(url, output_path)
    print(f'  Saved: {output_path}')
```

## 9. 新增封面任务工作流

1. **建目录**：`tasks/<任务名>/`
2. **写需求**：在 `quest.md` 中描述文案、情绪、动作（参考 `tasks/gaokao/quest.md`）
3. **写脚本**：复制 `tasks/gaokao/gen_cover_gaokao.py`，按第 6 节模板替换 prompt 中的动作/表情/标题（脚本默认一次跑出 4 个比例）
4. **生成图片**：`python3 tasks/<任务名>/gen_cover_<任务名>.py`，运行后会自动得到 3:4 / 4:3 / 16:9 / 2.35:1 四张封面
5. **验收**：检查人物一致性、文字准确性、构图、配色是否符合标准；若某个比例构图需要调整，再单独重跑该比例

## 10. 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| `FileNotFoundError: assets/face/...` | `BASE_DIR` 回退层数不够 | 使用 3 层 `os.path.dirname` |
| `image size must be at least 3686400 pixels` | size 太小 | 使用 ≥2560 宽度的尺寸 |
| 文字出现错字/乱码 | prompt 未强调文字准确性 | 在 prompt 中明确"无错字、无变形、无乱码" |
| 人物面部漂移 | 只传了单张参考图 / 未传参考图 | 传入 `assets/face/` 下全部图片作为 `image` 列表，模型综合参考 |
| 人物表情不像本人 | 未约束表情参考来源 | 在 prompt 中明确"表情符合参考图本人五官肌肉运动规律" |
| 多图传参报错 | `image` 参数格式错误 | 必须是 `list[str]`，每个元素为 `data:image/<fmt>;base64,...` |
